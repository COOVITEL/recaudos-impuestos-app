Recaudos Impuestos App

Documentación de la aplicación

Descripción

Esta aplicación está diseñada para gestionar y automatizar el proceso de recaudo de impuestos. Proporciona herramientas para registrar, consultar y analizar los datos relacionados con los impuestos de manera eficiente.

Características

  • Registro de contribuyentes y sus impuestos.
  • Consulta de historial de pagos.
  • Generación de reportes personalizados.
  • Interfaz amigable y fácil de usar.

Requisitos

  • Node.js y npm instalados.
  • Servidor web compatible.

Instalación

  1. Clona el repositorio: git clone https://github.com/usuario/recaudos-impuestos-app.git
  2. Accede al directorio del proyecto: cd recaudos-impuestos-app
  3. Instala las dependencias: npm install
  4. Configura las variables de entorno según el archivo env.d.ts.
  5. Inicia la aplicación: npm start

Uso

Una vez instalada, accede a la aplicación desde tu navegador en el puerto configurado (por defecto http://localhost:4000, o el puerto especificado en el archivo env.d.ts).

Guía del Proyecto

Estructura del Proyecto

  • APIs: Las peticiones principales están en la carpeta api/. Ejemplos:
    • peticionesTunja.ts: Maneja las peticiones relacionadas con la ciudad de Tunja.
    • registersRecaudos.ts: Gestiona los registros de recaudos.
  • Componentes: Los componentes reutilizables están en registros/components/. Ejemplos:
    • Barcode.tsx: Maneja la lectura de códigos de barras.
    • HeaderLote.ts: Genera encabezados para lotes de registros.
  • Páginas: Las vistas principales están en pages/. Ejemplos:
    • index.astro: Página principal.
    • operaciones.astro: Página para gestionar operaciones.
    • registros.astro: Página para consultar registros.
  • Utilidades: Funciones auxiliares están en utils/. Ejemplos:
    • JsonMessages.ts: Maneja mensajes en formato JSON.
    • state.ts: Gestiona el estado global de la aplicación.

Configuraciones Importantes

  • Puerto: El puerto de la aplicación se configura en env.d.ts. Por defecto, la aplicación corre en http://localhost:4000.
  • Tokens: Los tokens de autenticación se gestionan en las peticiones de la carpeta api/. Asegúrate de revisar los encabezados de las solicitudes.

Datos y APIs

  • Datos del código de barras: Los datos leídos del código de barras se procesan en registros/components/Barcode.tsx y se almacenan en la base de datos configurada.
  • APIs: Las rutas de las APIs están definidas en los archivos de la carpeta api/. Ejemplo:
    • registersRecaudos.ts: API para gestionar registros de recaudos.

Errores Comunes

  • Base de datos no conectada: Verifica que el servicio de la base de datos esté corriendo y que las credenciales en env.d.ts sean correctas.
  • Token inválido: Asegúrate de que el cliente envíe el token correcto en el encabezado Authorization.
